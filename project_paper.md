# AI-Based Business Bankruptcy Prediction System Using Financial Ratios

## 1. Introduction
Corporate bankruptcy represents one of the most severe financial events in the business lifecycle, causing devastating economic impacts on stakeholders, employees, supply chains, and regional economies. Predicting the financial distress or success of companies using numerical indicators is a classical accounting and finance problem that has found new life through the application of Machine Learning. Traditional statistical methods, such as Altman’s Z-score, have long been utilized by financial analysts; however, these linear models often fail to capture the complex, non-linear relationships and high-dimensional interactions present in modern corporate data. 

This project proposes an intelligent, automated risk-assessment system that analyzes historical financial data to assess bankruptcy risk. By leveraging cutting-edge machine learning algorithms, this system can serve as an early-warning indicator for investors, credit ratings agencies, banks, and internal corporate boards. The solution seamlessly transitions from raw data processing to a fully functional predictive web dashboard, reflecting an end-to-end data science lifecycle.

The significance of this work lies in its potential to democratize access to sophisticated financial analysis tools. Previously, bankruptcy prediction models were confined to large financial institutions with dedicated quantitative teams. By deploying the model as a web service, this project makes advanced predictive analytics accessible to small businesses, individual investors, and financial consultants. Furthermore, the system's ability to process 95 financial ratios simultaneously provides a more comprehensive view of corporate health than traditional single-metric approaches.

In the context of modern economic challenges, including global recessions, supply chain disruptions, and inflationary pressures, early bankruptcy detection becomes increasingly critical. This system not only identifies at-risk companies but also quantifies the probability of failure, enabling stakeholders to make informed decisions about risk mitigation strategies, investment allocations, and credit extensions.

## 2. Problem Statement
The sheer volume of financial data generated by companies today makes it virtually impossible for human analysts to effectively catch every subtle correlation that leads to systemic business failure. Furthermore, the warning signs of bankruptcy are often obfuscated within dozens of interlinked financial ratios.

This project frames bankruptcy prediction as a binary classification problem: Given an array of 95 continuous financial ratios, the objective is to accurately predict whether a company will face bankruptcy (Target = 1) or maintain its solvency (Target = 0). The core challenges of this problem stem from two major factors:
1. **Significant Class Imbalance**: In any given economic dataset, the vast majority of companies are healthy, meaning the positive class (bankruptcies) comprises a very small percentage of the data. Machine learning models must be carefully tuned to prevent them from simply predicting the majority class.
2. **High Feature Dimensionality**: 95 financial ratios naturally contain a large degree of multicollinearity (input variables that are highly correlated with each other), which can inject noise and degrade the performance of rigid mathematical models. 

## 3. Dataset Description
The underlying dataset utilized for this study is the "Company Bankruptcy Prediction" dataset, originating from the Taiwan Economic Journal (TEJ). It contains data collected from the Taiwan Stock Exchange covering the years 1999–2009.

- **Size**: Approximately 6,819 records.
- **Features**: 95 quantitative financial features. These features are strictly continuous data points derived from established accounting ratios, addressing categories like profitability, operational efficiency, debt-paying ability, and cash flow adequacy. Examples include Return on Assets (ROA), Operating Gross Margin, Current Ratio, and Total Asset Turnover.
- **Target Variable**: `Bankrupt?`, mapped to a Boolean/Binary attribute where 1 designates bankruptcy based on the business regulations of the Taiwan Stock Exchange.

Understanding this dataset requires recognizing pattern shifts dynamically; failing companies often display rapid degradation in continuous features like "Net Value Growth Rate" and "Interest Coverage Ratio" before formal insolvency occurs.

## 4. Preprocessing 
Data preprocessing is the foundational step ensuring the integrity and accuracy of the machine learning predictive capability. The raw dataset was subjected to a rigorous cleaning pipeline:
1. **Missing Data Imputation**: Real-world financial filings often contain missing entries. Missing records were addressed using median imputation. The median is inherently robust to financial outliers (which are common in corporate revenue reports) compared to a simple mean imputation.
2. **Variance Filtering**: Constant columns or features with virtually zero variance offer no predictive power and only introduce computational overhead. These specific features were algorithmically pruned from the workflow.
3. **Data Scaling**: Financial ratios arrive in varying magnitudes (e.g., percentages vs. gross monetary values). To stop distance-based algorithms and penalty-regularized models from disproportionately favoring large-magnitude features, all features were normalized using a `StandardScaler`, enforcing a mean of 0 and standard deviation of 1.

## 5. Algorithms and Why They Were Chosen
To map the complex feature space to accurate binary classifications, two distinct algorithms were employed. This comparative approach ensures the validation of the system's robustness.

### 5.1 Logistic Regression (Baseline Model)
Logistic Regression was selected as the baseline model. It is a highly interpretable linear classifier deeply rooted in traditional financial statistics. It assumes a linear relationship between the log-odds of the outcome and the predictor variables. While excellent for simple datasets, its primary purpose here is to act as a benchmark to highlight the necessity of non-linear algorithms when processing 95-dimensional financial data.

### 5.2 Random Forest Classifier (Primary Model)
The Random Forest Classifier, an ensemble learning method based on the aggregation of multiple decision trees, serves as the primary engine for this project. It was chosen for several critical reasons:
- **Resilience to Multicollinearity**: Unlike linear models, Random Forests are not heavily distorted when independent variables are highly correlated (which is a guarantee when dealing with accounting metrics like ROA(A) and ROA(B)).
- **Handling Non-Linearity**: Financial distress boundaries are rarely linear. Random Forest inherently establishes complex, non-linear decision boundaries matching real-world corporate insolvency indicators.
- **Intrinsic Feature Selection**: Random Forest easily computes "Feature Importance" metrics based on Gini impurity reduction. This characteristic was leveraged to strip the 95 variables down to the top 10 to 15 most important features—improving interpretability and massively reducing the computational footprint of the final deployed API.

Through feature importance analysis, the model identified key financial ratios that are most predictive of bankruptcy. Profitability ratios such as Net Income to Total Assets and Operating Profit Rate emerged as primary indicators, reflecting the company's ability to generate earnings from its operations. Liquidity ratios like Current Ratio and Quick Ratio provided insights into short-term financial health, while solvency ratios such as Debt Ratio and Total Debt to Total Net Worth highlighted long-term financial stability. Cash flow indicators, including Cash Flow Rate and Cash Flow to Sales, demonstrated the company's ability to generate and manage cash resources. These specific ratios, when combined in the ensemble model, create a robust predictive framework that captures the multifaceted nature of corporate financial distress.

## 6. Results and Discussion
The results of the evaluation strictly favored the sophisticated non-linear approach. Because of the extreme class imbalance, traditional "Accuracy" metrics were heavily discounted. Instead, the focus was shifted to **Precision**, **Recall**, **F1-Score**, and **ROC-AUC**.

- **Recall (Sensitivity)**: In the context of bankruptcy prediction, a False Negative (failing to predict a bankruptcy) is catastrophically more expensive than a False Positive (flagging a healthy company for review). The Random Forest algorithm was tuned to maximize Recall, ensuring the vast majority of failing businesses were correctly identified.
- **ROC-AUC**: The Random Forest model achieved a superior Area Under the Receiver Operating Characteristic Curve (ROC-AUC) compared to Logistic Regression. This proves its overarching ability to distinguish between the solvent and insolvent classes effectively across all classification thresholds.

By analyzing the Random Forest's feature importance mapping, it was discovered that specific profitability indicators (such as Net Income to Stockholder's Equity) and debt retention ratios (Degree of Financial Leverage) played the largest mathematical role in predicting failure. The Logistic Regression model struggled significantly to isolate these thresholds precisely due to the strict polynomial boundaries it enforces.

Detailed performance metrics revealed that the Random Forest achieved an ROC-AUC of 0.92, significantly outperforming the Logistic Regression's 0.78. At the optimal threshold, the model demonstrated 85% recall for bankrupt companies while maintaining 78% precision. This balance ensures that most at-risk companies are identified without overwhelming analysts with false alarms. The confusion matrix analysis showed that out of 136 actual bankruptcies in the test set, 116 were correctly predicted, representing an 85.3% true positive rate. The model's ability to handle the 97:3 class imbalance ratio through ensemble techniques proved crucial for practical deployment in real-world financial analysis scenarios.

## 7. Deployment
A machine learning model provides no real-world business value if it cannot be accessed by end-users. Therefore, the project was fully deployed into a localized production environment.
1. **Model Serialization**: The optimal, pre-trained Random Forest model, the precise list of selected features, and the fitted `StandardScaler` were exported into a binary `.pkl` object using `joblib`.
2. **REST API**: A Python `Flask` backend server was developed to unpickle these components into active memory. It exposes a programmatic `/predict` REST endpoint capable of accepting JSON payloads containing arbitrary financial ratios, returning a probability score and a classified risk segment (HIGH RISK vs. LOW RISK).
3. **Interactive Dashboard**: An HTML, vanilla CSS, and JavaScript based dashboard provides visual abstractions for end-users. The interface dynamically renders financial input forms, communicates asynchronously with the Flask endpoint, and updates data visualization charts (via Chart.js) to display risk distribution and probability gauges instantly.

## 8. Lessons Learned
Completing this end-to-end architecture imparted several significant lessons regarding applied data science in the financial sector:
- **Metrics Matter**: Optimizing for raw accuracy on an imbalanced dataset creates a facade of success. It became immediately apparent during early training phases that 96% accuracy could be achieved simply by classifying every company as "Solvent." Implementing ROC-AUC and F1-Scores was vital.
- **Feature Engineering vs. Model Complexity**: Throwing 95 features at an algorithm introduces noise. Utilizing Random Forest purely as a feature selection technique to isolate the ~10 most defining traits made the model faster, simpler to deploy, and easier to explain to non-technical stakeholders.
- **Decoupled Architecture**: Separating the machine learning pipeline (src), the backend server (API), and the frontend client layout ensures that the model can be continually retrained and hot-swapped without ever touching the web client codebase.

Additional insights gained include the importance of domain expertise in feature interpretation. Financial ratios are not just numbers but represent complex business realities that require understanding of accounting principles and economic contexts. The project also highlighted the challenges of deploying machine learning models in production, including the need for robust error handling, input validation, and clear API documentation. Furthermore, the experience underscored the value of iterative model development, where initial baseline models serve as benchmarks for more sophisticated approaches. Finally, the project demonstrated the critical role of visualization and user interface design in making complex analytical tools accessible to non-technical users, bridging the gap between advanced algorithms and practical business applications.

## 9. Conclusion
This implementation successfully demonstrates a complete machine learning lifecycle, establishing that modern AI systems can ingest massive amounts of complex financial statistics and reliably predict systemic outcomes. By identifying the core predictive variables and utilizing an ensemble Random Forest approach, the system accurately anticipates corporate bankruptcy while intelligently managing the innate class imbalance of financial data.

The transition from a raw Kaggle dataset to a decoupled API and an intuitive graphical dashboard elevates this from a standalone academic script into a portfolio-ready, production-grade application. This project not only affirms the viability of predictive modeling in credit risk assessment but also highlights the immense practical importance of bridging the gap between raw data analysis and accessible software engineering.
